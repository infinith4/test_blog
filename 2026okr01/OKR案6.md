# OKR案6: 次世代PM/PL育成とAI時代のプロジェクト推進力

## コンセプト

**PMBOKに縛られない、実践的でモダンなPM/PLスキルアップ**

従来のPMBOK的な形式知よりも、実践を通じた暗黙知の獲得と、AI時代に求められる新しいプロジェクト推進力を重視。チーム全体の成長と事業拡大への貢献を中心に据えたOKRです。

**3つの重点領域:**
1. **実践的PM/PL力**: プロジェクトを実際に動かす力（計画より実行、ドキュメントより成果）
2. **AI時代のリーダーシップ**: AI活用をリードし、チームに展開する力
3. **事業貢献マインド**: 顧客価値創造と組織成長への貢献

## Objective 1: 実践的PM/PL育成 - 次世代リーダーの自律的成長

**背景**:
- PLの役割を担えるメンバーを育成し、PL経験者はPMとして一人立ちできるようフォロー
- PMが開発の下地を作るのではなく、中堅メンバーに権限委譲してタスクを任せる
- 座学ではなく、実プロジェクトでの経験を通じた成長を重視

### KR1-1: PM/PLタスクの段階的移譲率80%達成

**指標**: 現在PMが担当しているタスクのうち80%を中堅メンバー（PL候補、PM候補）に移譲し、自律的に遂行できる状態にする

**移譲対象タスク（現状PMが実施している業務）:**

#### フェーズ1: PLタスク（Q1-Q2）
1. **タスク分解とアサイン** (20%)
   - WBS作成、タスクの優先順位付け、メンバーへの割り振り
   - 現状: PM主導 → 目標: PL候補が主導、PMは週1レビュー

2. **進捗管理と障害除去** (20%)
   - デイリースタンドアップのファシリテーション
   - ブロッカーの特定と解決
   - 現状: PM主導 → 目標: PL候補が主導、PMは緊急時のみ介入

3. **品質担保とレビュー** (20%)
   - コードレビューのリード、品質チェックリスト運用
   - テスト計画策定と実行管理
   - 現状: PM主導 → 目標: PL候補が主導、PMはスポットレビュー

#### フェーズ2: PMタスク（Q3-Q4）
4. **顧客コミュニケーション** (20%)
   - 週次定例での進捗報告、課題共有
   - 仕様変更の交渉、受け入れ条件の調整
   - 現状: PM主導 → 目標: PM候補が主導、PMは同席サポート

5. **リスク管理と意思決定** (20%)
   - プロジェクトリスクの識別と対応策立案
   - スコープ/スケジュール/リソースの調整判断
   - 現状: PM主導 → 目標: PM候補が提案、PMは最終承認のみ

**測定方法:**
- 月次での移譲タスク実施率測定（自己評価 + PMレビュー）
- 各タスクのオーナーシップ度合いを5段階評価
  - Lv1: PMが全て実施
  - Lv2: PM主導、メンバーがサポート
  - Lv3: メンバー主導、PMがレビュー
  - Lv4: メンバーが自律実施、PMは週次確認のみ
  - Lv5: メンバーが完全自律、PMは月次確認のみ
- 目標: 全タスクでLv4以上達成 = 移譲率80%

**具体的アクション:**
- Q1: 各メンバーの現状スキルマップ作成、育成計画策定
- Q2: PLタスク（1-3）の段階的移譲開始、週次1on1でのOJT
- Q3: PLタスクの自律化確認、PMタスク（4-5）の移譲開始
- Q4: 全タスクの自律化確認、次年度の完全委譲計画策定

**成功の定義:**
- 中堅メンバー2-3名が、PMの介入なしで小規模プロジェクト（2-3ヶ月、3-5名規模）をリードできる状態
- PMが別プロジェクトに並行参画できる余力創出（現在のPM工数30%削減）

### KR1-2: プロジェクト成果の質的向上 - 顧客満足度4.5/5.0以上

**指標**: プロジェクト完了時の顧客満足度アンケート結果で4.5/5.0以上を達成

**評価項目（各5段点）:**
1. **成果物の品質**: バグの少なさ、パフォーマンス、使いやすさ
2. **納期遵守**: 予定通りの納品、遅延時の早期報告と対応
3. **コミュニケーション**: レスポンスの速さ、わかりやすい説明、透明性
4. **柔軟性**: 変更要望への対応、代替案の提案
5. **プロフェッショナリズム**: 技術力、問題解決力、提案力

**現状値**: 4.0/5.0（推定） - 品質は良いが、コミュニケーションや柔軟性に改善余地
**目標値**: 4.5/5.0以上（全項目で4点以上、うち3項目で5点）

**具体的アクション:**
- プロジェクト開始時に顧客の期待値を明確化（キックオフで確認）
- 週次定例での積極的な進捗可視化（デモ実施、リスク早期共有）
- 変更要望発生時の迅速な影響分析と代替案提示（24時間以内）
- プロジェクト完了時の振り返り（KPT形式）と改善アクション

**測定方法:**
- プロジェクト完了後1週間以内にアンケート実施
- 四半期ごとの平均スコア算出とトレンド分析
- 低評価項目の原因分析と改善アクション策定

### KR1-3: チーム心理的安全性スコア4.0/5.0以上達成

**指標**: Googleの心理的安全性尺度に基づくチームアンケートで4.0/5.0以上を達成

**背景**:
- 若手が「わかりやすく教えてほしい」と期待している
- PM/PLが育成されるには、失敗を恐れず挑戦できる環境が必要
- AI時代のチームには、オープンなコミュニケーションとコラボレーションが不可欠

**心理的安全性の7つの指標（各5段階評価）:**
1. **リスクテイク**: 失敗を恐れずに新しいことに挑戦できる
2. **意見表明**: 異なる意見や懸念を自由に言える
3. **ヘルプシーキング**: わからないことを気軽に質問できる
4. **失敗からの学び**: ミスをしても責められず、学びの機会として捉えられる
5. **尊重**: メンバー同士が互いのスキルや貢献を尊重している
6. **透明性**: 情報がオープンに共有され、隠し事がない
7. **サポート**: 困ったときに助け合える文化がある

**現状値**: 3.5/5.0（推定） - 基本的には良好だが、さらなる向上余地あり
**目標値**: 4.0/5.0以上（全項目で3.5点以上、うち5項目で4点以上）

**具体的アクション:**
- 月次1on1での個別フォロー（全メンバー）
  - キャリア目標、不安や悩み、チャレンジしたいこと
  - フィードバックの双方向化（メンバーからPMへのフィードバックも歓迎）
- 週次振り返り会（KPT形式）
  - Keep: うまくいったこと、続けたいこと
  - Problem: 課題、困ったこと
  - Try: 次週試したいこと、改善アクション
- 失敗を共有する文化づくり
  - 「今週の失敗」共有タイム（学びに焦点）
  - ポストモーテム（事後分析）の実施（blame freeで原因究明）
- AI活用の失敗談も積極的に共有
  - 「AIに騙された話」「プロンプトの失敗事例」

**測定方法:**
- 四半期ごとに匿名アンケート実施
- スコアのトレンド分析と改善アクション策定
- 低スコア項目の深堀り（1on1でのヒアリング）

## Objective 2: AI時代のプロジェクト推進力 - 技術フラグシップの実現

**背景**:
- 生成AI事業の拡大、技術フラグシップ（LangGraph、MCP、エージェント）
- コーディングエージェント、SDDを駆使した次世代の開発スタイル確立
- 生産性向上50%の実現

### KR2-1: AI駆動開発の実践率90%以上

**指標**: 開発タスクの90%以上でAIツール（Copilot、Cursor、ChatGPT/Claude、LangGraph）を実質的に活用

**実践率の定義:**
- 実践率 = AI活用タスク数 / 全開発タスク数
- AI活用タスク = コード生成、レビュー、テスト作成、ドキュメント作成、設計支援などでAIツールを使用し、生産性向上に寄与したタスク

**AI活用の3つのレベル:**

#### レベル1: 基本活用（全メンバー必須）
- **GitHub Copilot/Cursor**: リアルタイムコード提案、補完
- **ChatGPT/Claude**: コード説明、バグ調査、簡単なコード生成
- **活用時間**: 開発時間の30%以上
- **成果**: コーディング速度20-30%向上

#### レベル2: 実践活用（PL/PM候補が習得）
- **AIペアプログラミング**: Cursorのコンポーザー、Copilot Chat活用
- **テスト自動生成**: AIによるユニットテスト、E2Eテスト生成
- **ドキュメント自動生成**: README、API仕様書、設計書の自動生成
- **活用時間**: 開発時間の50%以上
- **成果**: 開発工数30-40%削減

#### レベル3: 先進活用（技術フラグシップ）
- **LangGraph/MCP活用**: エージェント開発、自動化タスク実装
- **SDD（Specification-Driven Development）**: 仕様からのコード自動生成
- **AIレビュー**: AIによるコードレビュー支援、品質チェック
- **活用時間**: 開発時間の70%以上
- **成果**: 開発工数50%削減、新技術のプロトタイプ構築

**測定方法:**
- 週次タスク完了時に「AI活用度」を自己申告（レベル1-3）
- GitHub Copilot/Cursor使用ログの分析
- 月次での活用事例共有会（成功例、失敗例）

**具体的アクション:**
- Q1: AI活用ガイドライン作成、プロンプトテンプレート集整備
- Q1-Q2: レベル1の全員習得、レベル2のPL/PM候補習得
- Q2: AI活用事例DB構築（成功例、失敗例、ベストプラクティス）
- Q3-Q4: レベル3の実践、新技術（LangGraph、MCP）のプロトタイプ開発
- 月次AI勉強会開催（最新ツール紹介、活用Tips共有）

**成功の定義:**
- 全メンバーがレベル1を習得（活用率100%）
- PL/PM候補2-3名がレベル2を習得（活用率80%以上）
- チーム全体でレベル3のプロトタイプを1つ以上構築

### KR2-2: 生産性50%向上の実証（工数削減ベース）

**指標**: AI活用前後のプロジェクト工数比較で、平均50%の工数削減を実証

**測定方法:**
- **ベースライン**: 直近3プロジェクトの平均工数（AI活用前または低活用）
- **改善後**: AI全面活用後のプロジェクト工数（平均値）
- **比較**: (ベースライン工数 - 改善後工数) / ベースライン工数 × 100%

**工程別削減目標:**

| 工程 | AI活用前 | 削減率 | AI活用後 |
|------|---------|-------|---------|
| 要件定義・設計 | 20% | 30%削減 | 14% |
| コーディング | 40% | 50%削減 | 20% |
| テスト | 25% | 50%削減 | 12.5% |
| ドキュメント作成 | 15% | 70%削減 | 4.5% |
| 合計 | 100% | 約50%削減 | 約51% |

**具体的アクション:**
- **要件定義・設計**:
  - ChatGPT/Claudeでの要件整理、ユーザーストーリー生成
  - 設計ドキュメントの自動生成、アーキテクチャ図の作成支援

- **コーディング**:
  - Copilot/Cursorでのコード生成、リファクタリング支援
  - AIペアプログラミングの実践

- **テスト**:
  - AIによるテストコード自動生成（カバレッジ80%以上）
  - テストデータ自動生成、E2Eシナリオ生成

- **ドキュメント作成**:
  - README、API仕様書、運用手順書の自動生成
  - コードからのドキュメント逆生成

**測定頻度:**
- プロジェクト開始時: ベースライン工数見積もり（AI活用なしと想定）
- プロジェクト完了時: 実績工数測定（AI活用度合いも記録）
- 四半期ごとの集計と分析

**成功の定義:**
- 年間2プロジェクト以上で50%工数削減を達成
- 工程別削減率の達成（コーディング50%、テスト50%、ドキュメント70%）
- AI活用ベストプラクティス集の完成（20事例以上）

### KR2-3: 技術的チャレンジの成功（新技術プロトタイプ構築）

**背景**:
- 技術フラグシップとしてのポジション確立
- 生成AIビジネス、未開の技術を切り開く
- 新しい技術のキャッチアップ、若手の模範

**プロトタイプの要件:**
1. **技術的新規性**: LangGraph、MCP、エージェント技術のいずれかを活用
2. **実用性**: 顧客への提案材料、または社内業務改善に繋がる
3. **チーム参画**: PM/PL候補がメイン開発者として参画
4. **成果物**: 動作するデモ + ドキュメント + 技術ブログ記事

**プロトタイプ例:**
- **エージェント型コード生成ツール**: LangGraphで要件から設計・実装まで自動化

- **ドキュメント生成エージェント**: コードから包括的なドキュメントを自動生成
- **プロジェクト管理AI**: リスク分析、進捗予測、提案自動生成

**測定方法:**
- 四半期ごとのプロトタイプ構築状況レビュー
- デモ実施とフィードバック収集（社内、顧客）
- 技術ブログ記事の公開（外部発信）

**具体的アクション:**
- Q1: 技術調査、プロトタイプテーマ選定（チーム全体でブレスト）
- Q2: プロトタイプ1構築開始、技術検証
- Q3: プロトタイプ1完成、デモ実施、プロトタイプ2構築開始
- Q4: プロトタイプ2完成、顧客提案または社内展開

**成功の定義:**
- 2つ以上のプロトタイプ完成（動作デモ可能）
- 顧客提案1件以上、または社内活用開始
- 技術ブログ記事2本以上公開（外部発信による技術力アピール）
- PM/PL候補が新技術をリードして習得

## Objective 3: 事業貢献マインドの醸成 - 組織成長への主体的関与

**背景**:
- 事業拡大や組織の成長を念頭に日々のプロジェクト推進
- 事業拡大のための顧客への一言、具体的な提案
- 組織成長のための若手への経験提供、これら一つ一つの積み重ねが今を作っている

### KR3-1: 顧客への提案・追加案件創出2件以上

**指標**: プロジェクト実施中または完了後に、顧客への新規提案または追加案件を2件以上創出

**提案の種類:**
1. **技術的改善提案**: 既存システムの改善、新技術導入
2. **機能追加提案**: AI機能の追加、ユーザー体験向上
3. **業務効率化提案**: 業務プロセスの自動化、AI活用支援
4. **新規案件提案**: 全く新しいプロジェクトのアイデア提案

**測定方法:**
- 提案内容と顧客反応を記録（Notion/Jira）
- 受注に至った場合は「成功案件」として特記
- 未受注でも提案実施は評価対象

**具体的アクション:**
- プロジェクト中の顧客との会話で課題をヒアリング
- 週次定例で小さな改善提案を積極的に実施
- プロジェクト完了時の振り返りで次のステップを提案
- 新技術プロトタイプを顧客にデモし、活用可能性を探る

**成功の定義:**
- 提案2件以上実施
- うち1件以上が顧客に好評（具体的検討に進む、または受注）

### KR3-3: 後継者育成の成功 - 次世代PM/PL候補の具体的成長

**指標**: 中堅メンバー2-3名が以下の成長を達成

**成長の定義（3つの観点）:**

#### 1. タスク遂行力の向上
- **現状**: PMの指示待ち、タスク分解が不十分
- **目標**: 自らタスクを分解し、優先順位をつけて進められる
- **測定**: タスク分解の質（レビュー評価）、自律実施率80%以上

#### 2. 問題解決力の向上
- **現状**: 問題発生時にPMに相談、解決策はPMが提示
- **目標**: 自ら問題を分析し、複数の解決策を提案できる
- **測定**: 問題解決の提案回数（月5回以上）、提案の実行率70%以上

#### 3. 他者育成力の獲得
- **現状**: 自分の作業に集中、他メンバーの支援は限定的
- **目標**: 若手メンバーの相談に乗り、技術的アドバイスができる
- **測定**: 若手からの評価4.0/5.0以上（四半期アンケート）

**具体的アクション:**
- Q1-Q2: タスク遂行力の向上（PLタスクのOJT）
  - タスク分解のレビュー、1on1でのフィードバック
  - 小規模タスクのリード経験を積ませる

- Q2-Q3: 問題解決力の向上（PMタスクのOJT）
  - 問題発生時に「どうしたらいい?」ではなく「こうしたいがどうか?」を引き出す
  - 解決策の提案を促し、一緒に検討（答えを教えるのではなく、考えさせる）

- Q3-Q4: 他者育成力の獲得
  - 若手メンバーのメンターとして任命
  - コードレビューのリード、技術相談対応を任せる

**測定方法:**
- 月次1on1での進捗確認（自己評価 + PMレビュー）
- 四半期ごとの360度評価（上司、同僚、後輩からの評価）
- タスク実施記録の分析（自律率、問題解決提案数）

**成功の定義:**
- 2-3名のPM/PL候補が、小規模プロジェクト（2-3ヶ月、3-5名）をリードできる
- 若手メンバーからの評価4.0/5.0以上
- 次年度、完全に権限委譲できる状態を実現
